## **AURA OS** 

Agentic Unified Runtime Architecture — операционная среда для команды ИИ-агентов 

Техническая документация 

Версия 1.0.0 · Июнь 2026 

## **1. Что такое AURA OS** 

AURA OS — настольное приложение для Windows, которое превращает разрозненные ИИ-инструменты в единую управляемую команду. Пользователь ставит задачу текстом или голосом, а система сама раскладывает её на подзадачи, распределяет их между подключёнными агентами с учётом их навыков, запускает агентов параллельно, организует перекрёстную проверку результатов и сохраняет всю накопленную информацию в общую память на базе Obsidian. 

Типовой сценарий: Claude Code пишет программный код, Codex проверяет его и находит ошибки, затем они совместно устраняют замечания — на выходе пользователь получает работающее приложение, вычищенное от ошибок. Тот же конвейер применим к любой задаче: исследования, документы, анализ данных, автоматизация. 

## **2. Архитектура** 

Система состоит из четырёх слоёв: 

|**Слой**|**Модуль**|**Назначение**||
|---|---|---|---|
|Интерфейс|renderer/|Современный тёмный UI: постановка задач (текст/голос), доска задач с живыми логами, карточки агентов, просмотр памяти, настройки. Два языка: RU/EN.||
|Оркестратор|src/orchestrator.js|Конвейер PLAN → EXECUTE → REVIEW/FIX → MEMORY. Декомпозиция задачи, маршрутизация по навыкам, параллельный запуск, циклы исправления.||
|Агенты|src/agents.js|Реестр агентов, автообнаружение установленных CLI, запуск|процессов, потоковая передача вывода, отмена.|
|Память|src/memory.js|Общая память в хранилище Obsidian: заметки по каждой задаче, сводный индекс Memory.md, контекст для планирования новых задач.||



Поставляются две сборки с одним и тем же ядром: standalone-исполняемый файл «AURA OS.exe» (локальный сервер + окно приложения, без зависимостей) и Electron-версия (папка aura-os/, запуск: npm install && npm start). 

## **3. Конвейер выполнения задачи** 

Шаг 1 — Планирование. Агент-координатор (настраивается, по умолчанию Claude Code) получает задачу, список доступных агентов с их навыками и выдержку из общей памяти. Он возвращает JSON-план: подзадачи, исполнители, зависимости. Если план получить не удалось, применяется базовый сценарий «исполнитель + ревьюер». 

Шаг 2 — Исполнение. Подзадачи без взаимных зависимостей запускаются параллельно (лимит настраивается). Зависимые подзадачи получают результаты предыдущих шагов в свой контекст. Все агенты работают в общей рабочей папке и видят файлы друг друга. 

Шаг 3 — Ревью и исправление. Подзадачи с ролью reviewer проверяют результат. Если ревьюер не ответил APPROVED, оркестратор запускает раунд исправления у исполнителя и повторное ревью (количество раундов настраивается). 

Шаг 4 — Память. Итог записывается в хранилище Obsidian: отдельная заметка по задаче (выводы каждого агента) и строка в сводном индексе. Эта память 

подмешивается в контекст планирования следующих задач — система «помнит» прошлый опыт. 

Субагенты. Агенты уровня Claude Code и Codex умеют самостоятельно порождать собственных субагентов внутри своей подзадачи — AURA OS этому не препятствует и собирает их совокупный вывод. 

## **4. Подключение агентов** 

Предустановлены профили пяти агентов. Агент считается доступным, если его команда найдена в PATH. 

|**Агент**|**Команда**|**Роли**|**Установка**|
|---|---|---|---|
|Claude Code|claude|координатор, кодер, ревь|юер<br>npm i -g @anthropic-ai/claude-code|
|Codex CLI|codex|кодер, ревьюер|npm i -g @openai/codex|
|Gemini CLI|gemini|кодер, исследователь|npm i -g @google/gemini-cli|
|Hermes Agent|hermes|исследователь, кодер|согласно документации Nous Research|
|Ollama (локальн|о)ollama|кодер, без интернета|ollama.com — любая локальная модель|



Свой агент. «Агенты» → «Добавить агента»: укажите команду, аргументы с плейсхолдером {prompt} (и {model} при необходимости), навыки и роли. Так подключается любой CLI-инструмент или локальная модель. Профили встроенных агентов также можно переопределять, а ненужных агентов — выключать переключателем. 

## **5. Общая память (Obsidian)** 

В Настройках укажите путь к вашему Obsidian vault. AURA OS создаст в нём папку AURA, где будет вести: (1) заметку на каждую выполненную задачу с выводами всех агентов и (2) сводный файл Memory.md — журнал, который агенты используют как долговременную память. Заметки связаны wiki-ссылками и доступны для просмотра как из Obsidian, так и из вкладки «Память» в самом приложении. 

## **6. Голосовой ввод** 

Кнопка микрофона использует браузерное распознавание речи, если оно доступно. Гарантированный способ на любой Windows 10/11 — системная диктовка: поставьте курсор в поле задачи и нажмите Win+H. 

## **7. Установка и запуск** 

Standalone: распакуйте AURA-OS-v1.0-win64.zip и запустите «AURA OS.exe». Откроется служебное консольное окно (журнал ядра) и окно приложения. Файл не подписан — при первом запуске SmartScreen попросит подтверждение («Подробнее» → «Выполнить в любом случае»). 

Electron (для разработчиков): в папке aura-os выполните npm install, затем npm start. Сборка установщика: npx electron-packager . "AURA OS" --platform=win32 --arch=x64. Требования: Windows 10/11 x64; установленный CLI хотя бы одного агента; Obsidian — опционально; для голосового ввода — микрофон.

## **9. Безопасность** 

AURA OS не хранит и не передаёт API-ключи: каждый агент использует собственную аутентификацию своего CLI. Приложение слушает только локальный адрес 127.0.0.1 и недоступно из сети. Агенты исполняют команды с правами пользователя — выдавайте им рабочую папку, а не системные каталоги. Чтение заметок памяти ограничено пределами выбранного vault.
